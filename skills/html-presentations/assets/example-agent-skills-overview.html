What Are Agent Skills?

A simple, open format for giving agents new capabilities and expertise.

agentskills.io · GitHub

Definition

Agent Skills are folders of instructions, scripts, and resources that agents can discover and use to do things more accurately and efficiently.

At its core: a skill is a directory containing a SKILL.md file — metadata plus Markdown instructions that tell an agent how to perform a specific task.

Skills can also bundle scripts/, references/, and assets/.

Why Agent Skills?

Agents are capable but often lack the context to do real work reliably.

For authors

Build capabilities once, deploy across multiple agent products.

For agents

Extend capabilities based on the task; load skills on demand.

For teams

Capture organizational knowledge in portable, version-controlled packages.

What Can Skills Enable?

  • Domain expertise — legal review, data pipelines, specialized workflows
  • New capabilities — creating presentations, building MCP servers, analyzing datasets
  • Repeatable workflows — multi-step tasks that are consistent and auditable
  • Interoperability — reuse the same skill across Cursor, Claude Code, VS Code, and other compatible tools

When Skills Are Used & What They Produce

The coding agent uses skills in two main ways.

  • Editing and executing code — One common case: the agent uses a skill while it is editing, manipulating, or executing source code (e.g. following a review workflow, running an EDA script, or iterating on an ML run). The skill guides how the agent behaves in that context.
  • Creating new things — Skills are also used to create new outputs: new scripts, HTML presentations, documents, configs, and so on. The agent follows the skill’s instructions to produce these.
  • Artifacts — Whatever a skill helps the agent produce — a slide deck, a Python script, a report, a diagram — is an artifact. Artifacts are the concrete results of using a skill.

Skill Structure

A skill is a directory. Minimum: SKILL.md. Optional: scripts, references, assets.

my-skill/
├── SKILL.md          # Required: YAML frontmatter + Markdown instructions
├── scripts/          # Optional: executable code (Python, Bash, etc.)
├── references/       # Optional: docs the agent reads when needed
└── assets/           # Optional: templates, data files, images

SKILL.md needs name and description in frontmatter; the body is free-form instructions.

Specification at a Glance

  • name — required; lowercase, hyphens; must match directory name; max 64 chars
  • description — required; what it does + when to use it; max 1024 chars
  • license, compatibility, metadata — optional
  • Body: Markdown instructions; no strict format; keep under ~500 lines, use references for detail

Full spec: agentskills.io/specification. Validate with skills-ref validate <path>.

Adoption & Open Development

Agent Skills are supported by many AI development tools (Cursor, Claude Code, VS Code, and others).

The format was originally developed by Anthropic, released as an open standard, and is adopted by a growing ecosystem. The standard is open to contributions.

Supplementary

Loading, Activation & Execution

  • Loading (discovery) — At startup, the harness scans configured skill directories. For each skill it parses only the frontmatter of SKILL.md (name, description) and injects that metadata into the agent’s system prompt (~50–100 tokens per skill). For filesystem-based agents, a path to SKILL.md is included so the model can request the file when needed.
  • Activation — When the user’s task matches a skill’s description, the agent (or harness) loads the entire SKILL.md into context. Keep the main file under ~500 lines; put long reference material in references/.
  • Execution — The agent follows instructions step by step. It may read files from references/, run scripts from scripts/, or use assets/ only when told. Script execution is typically via shell commands (e.g. uv run script.py).

What Is a Coding Agent Harness?

The agent harness is the software infrastructure that wraps around an LLM or AI agent — everything except the model itself.

It manages the lifecycle of context: intent capture, tool orchestration, execution, verification, persistence.

  • Context management — state across sessions, checkpointing
  • Tool orchestration — API calls, timeouts, retries
  • Error recovery — hallucinations, invalid responses, network failures
  • State persistence — save progress between sessions

Skills are loaded by the harness. The harness discovers skills in configured directories and injects their metadata (or full content) into the agent’s context.

Where Are Agent Skills Installed?

Skills live in directories that the harness (Cursor, Claude Code, etc.) scans at startup. Common locations:

Project-level

  • .cursor/skills/
  • .agents/skills/
  • .claude/skills/
  • .codex/skills/

User-level (global)

  • ~/.cursor/skills/
  • ~/.claude/skills/
  • ~/.codex/skills/

Each path contains skill folders (e.g. ~/.cursor/skills/html-presentations/ with SKILL.md inside). Exact set depends on the product.
